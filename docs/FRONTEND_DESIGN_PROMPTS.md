# Chaoke 前端全页面设计 Prompts（基于“百科 + 图文帖子”）

> 用途：把下面的 Prompt 复制到 Figma AI / 设计生成工具里，快速产出同一风格的高保真页面草图，再人工精修。
>
> 约束：
> - **主题色（附图1）**：`#3388FF`（主色）、`#46EBD5`（辅助）、`#D377CD`（点缀）、`#000000/#FFFFFF`（中性色）
> - **吉祥物（附图2）**：3D 黑色大眼猫（圆润、可爱、偏潮玩）
> - **整体风格参考其余附图**：明亮、卡片化、霓虹高亮、超大圆角、轻拟物阴影、信息密度适中、强标签体系、底部 Tab + 中央「+」发帖按钮

---

## 0) 全局前缀 Prompt（每个页面都要先粘贴这一段）

```text
你是一名顶级移动端 UI/UX 设计师，请为「潮客 Chaoke」设计 iOS 风格的高保真移动端界面（画布 375×812，白底浅色主题，支持安全区）。

品牌与风格：
- 主题色来自调色板：主色 #3388FF；辅助 #46EBD5；点缀 #D377CD；中性色 #000000/#FFFFFF。
- 视觉语言参考：卡片化信息流、霓虹高亮点缀、超大圆角（16–28）、柔和投影、简洁线性图标 + 少量渐变色块。
- 吉祥物：3D 黑色大眼猫（圆润潮玩），用于启动页/空状态/AI 区域/关键引导提示，不要喧宾夺主。

文案态度（必须贯彻在所有页面）：
- 面对游客：热情欢迎 + 真诚道歉 + 具体可执行的帮助（参考“淄博旅游大火”时那种不做作的真诚：不喊口号、先把人照顾好）。
- 面对志愿者：表达敬意与感谢，突出潮汕人“重情义、讲信用、做事利落”的精神；让游客感到“来了就是自己人、宾至如归”。
- 语气：不营销、不油腻、不硬煽情；句子短、真诚、落到行动（下一步按钮/入口清晰）。
- 常用短句（可在空态/错误态/引导里使用）：欢迎你来潮汕｜路上辛苦了｜给你添麻烦了，真诚抱歉｜有需要尽管说｜谢谢你愿意帮一把｜我们会尽快核实/补充。

通用组件规范：
- 顶部：大标题 + 搜索框（圆角胶囊）+ 右侧图标按钮（消息/筛选/更多）。
- 中部：信息卡片（大圆角 + 轻阴影 + 清晰分区），标签 chips（胶囊、可多选）。
- 底部：Tab Bar（帖子/百科/AI/我的），中央凸起「+」主按钮（发帖），选中态用主色高亮。
- 文案：中文为主，少量英文点缀；整体友好、轻松但可信赖（旅游保障要有“权威感”）。
- 交互：重要主按钮突出（主色或主色渐变），次按钮描边或浅底；错误/风控提示用更克制的对话框/Toast。

输出要求：
- 给出该页面的完整布局（模块、组件、层级、关键文案、状态），并包含：默认态 + 空态 + 错误态（若适用）。
- 图标/插画风格统一；避免出现任何水印/平台 Logo。
```

## 1) 页面 Prompts（复制“全局前缀” + 对应页面 Prompt）

### A. 启动与登录

#### A1. 启动页 / Splash（可选但推荐）

```text
页面：启动页 Splash
目标：建立品牌记忆与气质（潮流、可信、轻社交）。
布局：纯白背景 + 大面积渐变光晕（#3388FF→#46EBD5，少量 #D377CD 点缀）+ 中心 3D 黑猫吉祥物（坐姿，微光边缘）+ Logo「潮客 Chaoke」+ slogan「一起把潮汕讲清楚」。
细节：底部小字“本项目为首届‘北回归线极客节’支持项目”；加载进度条（细胶囊）用主色。
```

#### A2. 登录页 / Auth（微信 + 手机号）

```text
页面：登录/注册
布局：上半屏渐变背景（蓝→青），右上角漂浮小猫贴纸；中部卡片：微信一键登录（主按钮）、手机号验证码登录（次按钮）、隐私协议勾选；底部放“先随便逛逛（游客模式）”入口（弱化但可见）。
文案：突出“登录后可发帖/求助/举报/志愿对接”；在按钮下放一行真诚欢迎文案（例：欢迎你来潮汕，路上辛苦了；如果有招待不周，我们先说声抱歉）；协议区展示《用户协议》《隐私政策》《免责声明》。
状态：未勾选协议时按钮置灰；错误态 toast；空态用小猫提示气泡。
```

#### A3. 绑定手机号 / Bind Phone（可选，登录后补全）

```text
页面：绑定手机号
布局：顶部返回 + 标题“绑定手机号”；中部表单卡：手机号输入、验证码输入、获取验证码按钮（倒计时）、提交按钮（主色）。
说明卡：解释“用于联系志愿者/接单确认/安全风控”；隐私提示（手机号仅用于必要联系，展示脱敏）。
状态：验证码错误/超时 toast；成功页小猫点头“绑定成功”。
```

### B. 主导航（Tab：帖子 / 百科 / AI / 我的）

#### B1. 帖子流 / Feed（首页）

```text
页面：帖子流 Feed（默认 Tab）
目标：高效浏览“接送供需、价格举报/求证、避坑分享”等帖子。
布局：
- 顶部：大标题「帖子」+ 搜索框“搜：甘草水果/酒店加价/接送…” + 右侧筛选按钮。
- 标签 chips 横滑：#免费接送 #我能接 #我需要 #价格反馈 #举报 #求证 #避坑 #服务点…
- 信息流卡片（大圆角）：左侧作者头像+昵称+时间+地点；正文 2 行；媒体缩略图（1–3 张）；底部动作：点赞/评论/分享/收藏；右上角显示“状态 badge”（核实/需补证/已完成等），其余都用标签 chips 表达场景，减少“强类型感”。
- 底部 Tab + 中央「+ 发帖」按钮（凸起霓虹描边）。
空态：小猫拿放大镜 + 文案“欢迎你来潮汕｜先发一个求助/举报吧（如果信息不够，我们先说声抱歉，会尽快补上）”；错误态：重试按钮（按钮文案带轻道歉：抱歉网络开小差了）。
```

#### B2. 全局搜索 / Search（帖子 + 百科）

```text
页面：全局搜索
布局：顶部固定搜索框（自动聚焦）+ 取消；下方热搜标签（chips）；搜索结果区带分段 Tab：帖子/百科/标签。
帖子结果：沿用 Feed 卡片；百科结果：沿用 Wiki 卡片；标签结果：展示标签卡（关注按钮 + 关联数量）。
空态：小猫举牌“换个关键词试试”；错误态：网络重试。
```

#### B3. 标签聚合页 / Tag（百科 + 帖子混合）

```text
页面：标签聚合页（#甘草水果 / #房价 / #车价 等）
布局：顶部大标题“#标签名”+ 关注按钮；下方两个切换 Tab：帖子/百科；默认展示混合流：先百科卡片（置顶 1–2 条）再帖子列表。
百科置顶卡：展示“价格区间/注意事项/引用来源”摘要，右侧小猫指向重点。
```

### C. 帖子（Post）

#### C1. 帖子详情 / Post Detail（含动作区）

```text
页面：帖子详情
布局：顶部返回 + 分享；主体卡片：作者区、正文、媒体；下方“关键信息 cards（信息卡片）”模块：
- 用同一个 cards 组件渲染（标题 + 若干 key/value），不按“帖子类型”换组件，只是 cards 内容不同。
- 示例 cards：行程（起点→终点/时间窗/人数/集合点）｜价格（类别/金额/规格单位/证据说明）｜核实进度（状态与时间）。
动作区（底部 sticky）：
- 主按钮根据后端 `availableActions` 渲染（MATCH/CLAIM/ADD_EVIDENCE/COMPLETE/CANCEL…），按钮文案保持真诚克制（遇到失败要先道歉再给下一步）。
二级：取消、完成、举报不当、联系（占位）
附加：显示“动作时间线”（匹配/认领/取消/完成/爽约等）做可追溯。
```

#### C2. 发帖入口 / Create Post（统一入口）

```text
页面：发帖（统一入口，不做“强类型分流”）
目标：像小红书一样先写内容，再用“标签 + 信息卡片”把场景说明白；不同类别发帖尽量同一套体验。
布局：
- 顶部“发帖”+ 返回；右侧“发帖小贴士”按钮（点开是：隐私脱敏、证据上传、真诚说明、不要夸大）。
- 主体：正文输入框（可先写一句话），下方媒体上传（图/视频/音频），标签 chips（横滑 + 搜索添加）。
- 场景快捷 chips（不改变表单，只是帮用户一键填标签/信息卡片模板）：#免费接送 #价格反馈 #避坑 #服务点 #求证 #举报…
- 信息卡片区域（可折叠）：按钮“一键添加信息卡片”→ 弹出模板：行程卡片/价格卡片/证据说明/联系方式（占位）。
- 权限引导：当用户选择了“#我能接/#接送供给”相关标签时，如未认证志愿者，弹出温柔提示 + 引导去认证（文案要感谢与解释：为安全起见需要认证，麻烦你了，先说声抱歉）。
```

#### C3. 发帖表单 / Create Post Form

```text
页面：发帖表单（统一字段 + 信息卡片 cards）
通用字段：
- 标题（可选）、正文（建议必填）
- 标签选择（chips，多选；支持搜索添加；默认推荐：#免费接送/#价格反馈/#避坑/#服务点/#求证/#举报）
- 地点选择（地图/定位按钮；可选）
- 发生时间（可选；价格/事件类建议填）
- 媒体上传（图片/视频/音频；证据类建议至少 1 项）
信息卡片 cards（同一个组件，按模板自动建议但允许自由增删）：
- 行程卡片（接送）：起点→终点、时间窗、人数、集合点（可引用服务点百科）、备注；醒目提示爽约规则（语气真诚：为了不耽误彼此，辛苦配合）。
- 价格卡片（举报/求证）：类别（物价/房价/车价）、金额（可选）、计价方式/规格单位（可选）、证据说明；隐私脱敏提示（遮挡手机号/身份证/银行卡）。
- 求证卡片：一句话问题 + 场景字段（地点/时间/预算可选），提交后引导 AI/社区一起回答。
底部：预览按钮（次）+ 发布按钮（主）；发布前轻提示“内容会帮助更多游客，我们会认真对待；如有疏漏先说声抱歉，可随时补充证据/信息”。
```

### D. 百科（Wiki）

#### D1. 百科首页 / Wiki Index（搜索 + 分类 + 推荐）

```text
页面：百科首页
目标：像“事实库”，但保持潮流轻松。
布局：顶部大标题「百科」+ 搜索框；下方 3 个入口卡：价格与避坑 / 服务点与集合点 / 热门标签。
列表卡片：标题 + 标签 + 摘要 + “可引用”标识；价格类词条展示迷你价格区间条（蓝→青渐变）。
空态：小猫抱书 + “还没有词条，先从价格基准开始共建”。
```

#### D2. 百科详情 / Wiki Detail（嵌入相关帖子）

```text
页面：百科详情（例：甘草水果 / 酒店临时加价 / 出租车一口价）
布局：
- 头图：柔和渐变背景 + 词条标题 + 标签 chips + 小猫注释气泡（“重点看价格区间/避坑点”）。
- 正文：结构化分段卡片（概述/价格区间/常见套路/如何自证/相关链接）。
- 价格区间模块：用“区间条 + 常见规格列表”表现（单位/范围/时间说明），配“依据来源”引用卡片。
- 相关帖子嵌入：同标签/同关联词条的帖子流（优先已核实举报帖、接送相关供需帖），提供一键“发价格举报/求证帖”按钮。
- 可选：服务点词条展示小地图预览（卡片内）。
状态：无相关帖子时显示小猫引导发帖。
```

### E. 时间线与地图（都是视图页）

#### E1. 时间线页 / Timeline View

```text
页面：时间线（把带时间字段的百科按时间组合）
布局：顶部“时间线”+ 筛选 chips（节庆/表演/集市/巡游…来自标签）+ 日期选择；主体为纵向时间轴卡片，每条展示：时间、标题、地点、注意事项摘要；卡片可点进百科详情。
视觉：时间轴点用主色；每条卡片右上角小标签（类型）。
```

#### E2. 地图页 / Map View

```text
页面：地图（把带坐标的百科/服务点展示在地图上）

视觉：marker 用蓝/青/粉区分类别；面板卡片大圆角、轻阴影；选中态主色描边。
空态：小猫拿指南针。
```

### F. AI 向导（基于百科 + 已核实帖子）

#### F1. AI 页 / AI Guide & Price Query

```text
页面：AI 向导（查价/问答）
布局：
- 顶部：标题「问潮客」+ 小猫头像（3D）+ 右侧“引用规则”按钮（说明：只引用百科与已核实帖子）。
- 首屏建议问题卡片（圆角方块）：查甘草水果价格/酒店是否加价/车价是否合理/如何避免宰客。
- 对话区：气泡 + 引用卡片（必须展示来源：百科段落/核实举报帖），引用卡片可点开详情。
- 底部输入栏：输入框 + 语音（可选）+ 一键“发求证/举报帖”快捷按钮。
关键：当无依据时，AI 必须返回“不确定”，并引导发求证/举报帖（给出一键填充的草稿）。
```

### G. 我的（账号、志愿者、设置）

#### G1. 我的 / Me（含志愿者入口）

```text
页面：我的
布局：顶部头像+昵称+一句签名；下方 3 张快捷卡：
1) 我的帖子 2) 我的收藏 3) 我的消息/通知（可选）
志愿者模块：显示状态 badge（未认证/审核中/已认证/已暂停），主按钮“申请志愿者/进入志愿者中心”。
底部：设置、协议、免责声明；小猫作为小图标点缀。
```

#### G2. 我的帖子 / My Posts

```text
页面：我的帖子
布局：顶部标题“我的帖子”+ 筛选（全部/接送/举报/求证）；列表沿用 Feed 卡片，但增加状态 badge（已发布/已下架/待核实/已核实）。
空态：小猫递笔“发第一条帖子吧” + 按钮跳转发帖。
```

#### G3. 我的收藏 / Favorites

```text
页面：我的收藏
布局：顶部标题“收藏”+ Tab（帖子/百科）；列表卡片沿用对应样式；支持批量管理（编辑模式）。
空态：小猫抱心心“先去收藏一些有用的避坑信息吧”。
```

#### G4. 消息与通知 / Notifications（可选）

```text
页面：消息与通知
布局：顶部标题；列表卡片按类型分：系统通知（审核结果/风控提醒）、互动（评论/点赞）、接送动作（被匹配/被认领/取消/完成）。
每条：图标 + 标题 + 摘要 + 时间；未读用主色小圆点。
空态：小猫打盹“暂无消息”。
```

#### G5. 个人资料 / Profile

```text
页面：个人资料
布局：头像（可更换）、昵称、个性签名、常用城市（可选）；保存按钮（主色）；展示“对外公开资料预览”（卡片）。
风格：干净克制，但保持圆角与色彩点缀。
```

#### G6. 设置 / Settings

```text
页面：设置
布局：分组列表卡：账号与安全（绑定手机号、退出登录）、隐私（默认匿名、帖子可见性）、通用（清缓存、关于）、法律与声明（用户协议/隐私政策/免责声明）。
危险操作（退出登录/注销账号如有）使用红色文字但不刺眼，二次确认弹窗。
```

#### G7. 协议与免责声明 / Legal

```text
页面：用户协议 / 隐私政策 / 免责声明（3 个独立页面或同一容器页）
布局：顶部标题 + 目录锚点（可选）+ 正文排版（更适合阅读：行高、段落间距、重点高亮）。
底部：返回；若是首次登录必读，提供“同意并继续”（主色）。
```

#### G8. 志愿者申请 / Volunteer Apply

```text
页面：志愿者申请
布局：顶部说明卡（语气真诚，突出潮汕人的待客之道与讲信用：谢谢你愿意帮一把；也解释为什么需要认证：为了游客与志愿者双方安全，麻烦你了，我们先说声抱歉）；表单卡：姓名/联系方式/可服务时段/可接送路线（可选）；上传材料区（提示敏感信息将加密，仅审核员可见）。
提交后：成功页用小猫举牌“已提交，等待审核”，展示预计时间与可撤回按钮。
```

#### G9. 志愿者中心 / Volunteer Center（已认证）

```text
页面：审核工作台
```

#### H2. 待核实价格举报列表 / Moderation Price Queue

```text
页面：待核实价格举报列表
```

#### H3. 价格举报审核详情 / Moderation Price Detail

```text
页面：价格举报审核详情
```
