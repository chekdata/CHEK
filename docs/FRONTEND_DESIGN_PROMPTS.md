# CHEK 前端全页面设计 Prompts（MVP：有知 + 相辅）

> 用途：把下面的 Prompt 复制到 Figma AI / 设计生成工具里，快速产出同一风格的高保真页面草图，再人工精修。
>
> MVP 原则（不要过度设计）：
> - 先做一个真正能用的 **有知（Wiki）+ 相辅（帖子 + 评论）**。
> - **不做私信**、不做匹配/接单等复杂流程。
> - “信息卡片 cards”不做独立功能：后端只存正文 `body`，前端仅在渲染层可选把正文结构化块展示成卡片。
> - **AI来 / 劳热 / 辣辣嗦** 作为占位或后置（可设计一个“简单可用/敬请期待”的版本）。
>
> 约束：
> - **品牌英文名**：CHEK（不是 CHAOKE）
> - **主题色（附图1）**：`#3388FF`（主色）、`#46EBD5`（辅助）、`#D377CD`（点缀）、`#000000/#FFFFFF`（中性色）
> - **吉祥物（附图2）**：3D 黑色大眼猫（圆润、可爱、偏潮玩）
> - **整体风格参考其余附图**：明亮、卡片化、霓虹高亮、超大圆角、轻拟物阴影、信息密度适中、强标签体系、底部 Tab + 中央「+」发帖按钮
> - **潮汕话特色命名（UI 必须使用）**：
>   - 帖子：**相辅**
>   - 百科：**有知**
>   - AI：**AI来**
>   - 我的：**胶己**
>   - 时间线：**劳热**
>   - 地图：**辣辣嗦**

## 1) 页面 Prompts（复制“全局前缀” + 对应页面 Prompt）

### A. 启动与登录

#### A1. 启动页 / Splash（推荐）

```text
页面：启动页 Splash
目标：建立品牌记忆与气质（潮流、可信、轻社交）。
布局：纯白背景 + 大面积渐变光晕（#3388FF→#46EBD5，少量 #D377CD 点缀）+ 中心 3D 黑猫吉祥物（坐姿，微光边缘）+ Logo「潮客 CHEK」+ slogan「一起把潮汕讲清楚」。
细节：底部小字“本项目为首届‘北回归线极客节’支持项目”；加载进度条（细胶囊）用主色。
```

#### A2. 登录页 / Auth（微信 + 手机号）

```text
页面：登录/注册
布局：上半屏渐变背景（蓝→青），右上角漂浮小猫贴纸；中部卡片：微信一键登录（主按钮）、手机号验证码登录（次按钮）、隐私协议勾选；底部放“先随便逛逛（游客模式）”入口（弱化但可见）。
文案：突出“登录后可发相辅/评论/收藏”；按钮下放一行真诚欢迎+轻道歉文案（例：欢迎你来潮汕，路上辛苦了；如有招待不周，我们先说声抱歉）。
状态：未勾选协议时按钮置灰；错误态 toast；空态用小猫提示气泡。
```

#### A3. 绑定手机号 / Bind Phone（可选，登录后补全）

```text
页面：绑定手机号
布局：顶部返回 + 标题“绑定手机号”；中部表单卡：手机号输入、验证码输入、获取验证码按钮（倒计时）、提交按钮（主色）。
说明卡：解释“用于账号安全/找回”，隐私提示（展示脱敏）。
状态：验证码错误/超时 toast；成功页小猫点头“绑定成功”。
```

### B. 主导航（Tab：相辅 / 有知 / AI来 / 胶己）

#### B1. 相辅 / Feed（首页）

```text
页面：相辅（帖子流，默认 Tab）
目标：像小红书一样高效浏览帖子与评论热度。
布局：
- 顶部：大标题「相辅」+ 搜索框“搜：牌坊街/英歌舞/避坑/好吃的…” + 右侧筛选按钮。

- 标签 chips 横滑：#避坑 #吃啥 #住哪 #怎么玩 #交通 #亲子 #雨天 #性价比…
- 信息流卡片（大圆角）：作者头像+昵称+时间+地点（可选）；正文 2 行；媒体缩略图（1–3 张）；底部动作：点赞/评论/分享/收藏（可以都放，但评论按钮要最突出）。
- 底部 Tab + 中央「+ 发相辅」按钮（凸起霓虹描边）。
空态：小猫拿放大镜 + 文案“欢迎你来潮汕｜先发一条相辅问问大家吧”；错误态：重试按钮（按钮文案带轻道歉：抱歉网络开小差了）。
```

#### B2. 全局搜索 / Search（相辅 + 有知）

### C. 相辅（帖子 + 评论）

#### C1. 相辅详情 / Post Detail（评论为核心）

```text
页面：相辅详情

SEO/GEO 说明：在设计说明里给出 title/description/OG 占位；正文前可加一段“可引用要点”卡片（可选）。
```

#### C2. 来相辅入口 / Create Post

```text
页面：来相辅（统一入口）
目标：像小红书一样轻量发帖，不做复杂类型分流。
布局：
- 顶部“发相辅”+ 返回；右侧“发帖小贴士”按钮（点开：别暴露隐私、说清时间地点、尽量配图）。
- 主体：正文输入框（可先写一句话），下方媒体上传（图/视频，先选择再批量上传，禁止选中即自动上传）。
- 标签 chips：推荐 + 搜索添加（可选）；地点选择（可选）；时间（可选）。
- 底部：发布按钮（主色）+ 草稿（可选）。
```

#### C3. 发相辅表单 / Create Post Form（细化）

```text
页面：发相辅表单（MVP：轻量）
字段：
- 标题（可选）
- 正文（建议必填）
- 媒体上传（图片/视频）：支持多选预览、封面顺序调整（上移/下移）、取消选择、失败重试；点击“上传所选”或“发布”时统一上传。
- 标签（可选）
- 地点（可选）
- 发生时间（可选）

- “信息卡片”不做独立功能；如果正文里包含结构化块，展示端可以渲染成卡片，但发布端不需要专门做卡片编辑器。
- 话题添加交互参考小红书：正文里输入 `#` 触发候选话题列表，点选后回填到正文。
状态：发布中/loading；失败 toast（先道歉再给重试）；成功页引导去详情看评论。
```
