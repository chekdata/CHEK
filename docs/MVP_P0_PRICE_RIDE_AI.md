# MVP P0：有知（Wiki）+ 相辅（帖子 + 评论）

> 目标：不要过度设计，先做一个真正能用的 MVP：  
> - **有知**：潮汕旅游/文化的可检索“知识库”  
> - **相辅**：像小红书一样的帖子流，核心交互只保留**发帖 + 评论**  
>
> 明确不做：私信/聊天、匹配/接单、志愿者体系、复杂审核工作台、信息卡片独立功能。

## 1. 核心模型（P0 必须先定）

### 1.1 有知（Wiki）

- 面向“公开可索引”的内容页（利于 SEO/GEO）
- 字段尽量简单：标题/摘要/正文（Markdown）/标签/更新时间

### 1.2 相辅（Forum）

- 交互边界：**发帖 + 评论**（可加点赞/收藏为后续，不影响 P0）
- 信息卡片不单独设字段：后端只存正文 `body`；如果正文里包含结构化块（如 `chek-card` fenced block），前端可选渲染成卡片

## 2. 范围（P0）

### 2.1 有知（必须可用）

- 首页（推荐/分类/热门标签）
- 搜索（关键词 + 标签筛选）
- 词条详情（结构化排版 + 更新时间 + 相关相辅入口）

### 2.2 相辅（必须可用）

- 帖子流（列表）
- 发帖（标题可选 + 正文必填 + 图片/视频可选 + 标签可选 + 地点/时间可选）
- 帖子详情（正文 + 评论列表 + 发表评论）

### 2.3 账号（最小闭环）

- 登录（微信/手机号，按现有 auth-saas）
- 胶己（我的）：我的相辅、收藏（可选）、设置

## 3. 非范围（P0 不做 / 后置）

- **AI来**：可以先做占位页（P1 再做引用型问答）
- **劳热（时间线）/ 辣辣嗦（地图）**：P1（可先在有知首页放入口占位）
- 私信/聊天
- 匹配/接单/志愿者/审核工作台
- 信息卡片编辑器（不做额外功能；只把卡片当正文渲染能力）

## 4. 角色与权限（P0）

| 能力 | 游客（未登录） | 登录用户 | 管理员 |
|---|---:|---:|---:|
| 浏览有知/相辅 | ✅ | ✅ | ✅ |
| 搜索 | ✅ | ✅ | ✅ |
| 发相辅 | ❌ | ✅ | ✅ |
| 评论 | ❌ | ✅ | ✅ |
| 删除自己的相辅/评论 | ❌ | ✅ | ✅ |
| 编辑/发布有知 | ❌ | ❌（后置） | ✅ |

## 5. 核心流程（P0）

### 5.1 逛有知 → 去相辅问问（最常见）

1) 游客在有知搜索词条  
2) 进入词条详情，看“要点/注意事项/更新时间”  
3) 点击“发相辅问问大家”自动带上标签（例如 #牌坊街）  
4) 登录后发帖，帖子详情页承接评论

### 5.2 逛相辅 → 回到有知补知识（可选）

1) 游客刷相辅看到高质量帖子  
2) 详情页提示“相关有知词条”（可选聚合）  
3) 进入有知，形成“知识沉淀 + 讨论”的闭环

## 6. 数据结构（建议，围绕“有知 + 相辅”）

### 6.1 Wiki（有知）

- `wiki_entry`
  - `entry_id` / `slug`
  - `title` / `summary`
  - `body_md`（Markdown）
  - `tags[]`
  - `is_public/is_indexable`
  - `published_at/updated_at`

### 6.2 Post（相辅）

- `post`
  - `post_id`
  - `title?` / `body_md`
  - `tags[]`
  - `location_json`（可选）
  - `occurred_at`（可选）
  - `is_public/is_indexable`
  - `author_user_one_id`
  - `created_at/updated_at`
- `comment`
  - `comment_id` / `post_id`
  - `body`
  - `author_user_one_id`
  - `parent_comment_id?`（可后置）
  - `created_at`

> 信息卡片：不单独设字段，统一写进 `body_md`（后端不解析）。

## 7. 验收标准（P0）

1) 有知：可搜索、可打开详情页（公开 URL 可索引）。  
2) 相辅：可发帖、可看详情、可评论；无私信/无匹配等复杂功能。  
3) 统一正文存储：后端不为“信息卡片”单独建字段与功能。  
4) 基础体验：登录态可发布；未登录引导登录；错误态有克制的提示（先道歉再给重试）。
