# 「潮客 CHEK」PRD（v0.2，MVP：有知 + 相辅）

## 1. 背景与问题

潮汕旅游存在典型痛点：信息分散、避坑成本高、路线/玩法不好检索；同时潮汕民间文化（方言、仪式、节日、英歌舞流派、村寨习俗）“口述多、记录少、资料分散、难检索、难传承”。

**产品定位**：一个由本地与游客共同参与的潮汕内容平台，让游客更安心、更懂潮汕；让本地知识更容易被记录与传承。

**MVP 原则**：不要过度设计，先做一个真正能用的“百科 + 论坛”，其它能力后置。

## 2. MVP 目标与范围（P0）

### 2.1 MVP 两个模块（必须达成）

> UI 命名必须体现潮汕话特色（用户可感知）：
> - 百科 = **有知**
> - 帖子 = **相辅**
> - 我的 = **胶己**
> - AI = **AI来**（P1 可选）
> - 时间线 = **劳热**（P1 可选）
> - 地图 = **辣辣嗦**（P1 可选）

1) **有知（Wiki）**：可检索、可分享、可被搜索引擎抓取的知识库  
2) **相辅（Forum）**：像小红书一样的帖子流，核心交互只做 **发帖 + 评论**

### 2.2 明确不做（P0 禁止）

- 私信/聊天
- 匹配/接单/志愿者体系/审核工作台等重流程
- “信息卡片 cards”独立字段与功能（后端只存正文 `body`；不做卡片编辑器、不做卡片搜索/筛选）

## 3. 用户与角色（P0）

- 游客（未登录）：浏览、搜索
- 登录用户：发相辅、评论、删除自己的内容、收藏（可选）
- 管理员：下架内容；有知写入（MVP 可先只有管理员可写）

## 4. 功能需求（P0）

### 4.1 有知（Wiki）

- 有知首页：推荐/分类/热门标签（可简化成“搜索 + 热门词条”）
- 搜索：关键词 + 标签筛选
- 词条详情：
  - 结构化排版（小标题/列表/提示卡）
  - 固定“可引用块”：要点/注意事项/更新时间（利于 SEO/GEO）
  - 相关相辅：同标签帖子列表 + 一键“发相辅问问大家”

写入（MVP 先做最小）：

- 管理员创建/编辑/发布词条
- 版本回滚/多人协作后置

### 4.2 相辅（帖子 + 评论）

- 相辅帖子流：列表卡（作者/时间/地点可选/媒体缩略图/评论数）
- 发相辅：标题可选 + 正文必填 + 图片/视频可选 + 标签可选 + 地点/时间可选
- 相辅详情：
  - 正文 + 媒体
  - 评论区（核心）：评论列表 + 输入框（底部 sticky）
  - 点“评论”即聚焦输入框

评论（MVP 必做）：

- 发表评论
- 查看评论列表
- 回复二级（parentCommentId）可后置

内容安全（MVP 最小）：

- 举报入口（可后置为“反馈”）
- 管理员下架/删除
- PII 提示：发帖前提示不要暴露手机号/身份证等

### 4.3 胶己（我的）

- 我的相辅
- 收藏（可选）
- 设置（协议/隐私/免责声明/退出）

## 5. SEO + GEO（P0 必须考虑）

公开可索引页面：

- 有知详情：`/wiki/:slug`
- 相辅详情：`/p/:postId`（仅公开且允许索引）
- 标签页：`/tag/:tagSlug`

要求：

- SSR/SSG/ISR 输出语义化 HTML
- `title/description/canonical/og:*` 完整
- `sitemap.xml` / `robots.txt`
- JSON-LD：`Article` + `BreadcrumbList`（最小即可）

## 6. 里程碑（建议）

- **P0（先上线能用）**：有知（搜索+详情）+ 相辅（发帖+评论）+ 胶己（我的相辅/设置）+ 基础 SEO
- **P1（可选）**：AI来（引用型问答，占位→可用）、劳热/辣辣嗦（占位→最简可用）、收藏/点赞、回复评论

## 7. 成功指标（P0 上线 4–8 周内）

- 有知：有效词条数、搜索转化（搜索→进入词条）
- 相辅：发帖成功率、评论成功率、日活评论/帖比
- SEO：被收录页面数、自然搜索进入占比（有条件再接统计）
